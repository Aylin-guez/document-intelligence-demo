# Before / After

## Input

`sample_data.json`

Contains fictitious technical documentation data:

- system metadata
- system overview
- architecture layers
- modules
- workflow steps
- setup steps
- technical notes
- appendix notes

## Output

`output/technical_documentation_demo.pdf`

An eight-page professional technical documentation PDF.

## Simple explanation

The automation takes structured technical information and turns it into a clean PDF document.

Instead of manually formatting system descriptions, architecture notes, module tables, workflows, setup steps, and technical notes, the same structured input can generate consistent documentation.

## Commercial angle

Useful for software teams, startups, internal tools, API guides, onboarding manuals, project handoffs, and recurring technical documentation.
